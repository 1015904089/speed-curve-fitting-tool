# 05 换皮接口预留

## 目标

V1 中凡是非网格界面涉及图片、图标、装饰图、预览占位图等视觉资源的位置，都要预留换皮接口。

默认情况下使用 Qt 默认皮肤，不强制加载自定义皮肤。

## 范围

需要预留换皮接口：

- 按钮图标
- 滑块手柄图片或样式
- 结果区占位图片
- 运动预览占位图
- 折叠面板图标
- 状态提示图标
- 工具栏图标

不纳入换皮接口：

- 曲线区网格
- 坐标轴
- 曲线本身
- 控制点数据图形
- 封闭区域颜色

这些内容属于绘图功能本体，应由曲线绘制样式单独管理。

## 核心指导代码

V1 只需要预留接口，不需要实现完整皮肤系统。

```cpp
class SkinProvider {
public:
    virtual QIcon icon(const QString& name) const = 0;
    virtual QPixmap image(const QString& name) const = 0;
    virtual QString styleSheet(const QString& componentName) const = 0;
};
```

默认实现应尽量返回 Qt 默认外观：

```cpp
// DefaultSkinProvider:
// - 图标不存在时返回空 QIcon，让控件使用文字或 Qt 默认样式。
// - 样式表不存在时返回空字符串。
// - 图片不存在时返回空 QPixmap，由界面显示普通占位文本。
```

## 设计原则

- 默认不改变 Qt 原生控件外观。
- 后续换皮只替换资源和样式，不改业务逻辑。
- 换皮接口不能影响曲线网格绘制性能。
- 换皮失败不能阻塞软件启动。
