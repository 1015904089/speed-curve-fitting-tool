# 06 皮肤资源与预览解耦

## 目标

V2 中运动预览、按钮图标、状态提示和结果区视觉资源不能直接写死在业务逻辑中，应通过 V1 预留的换皮接口获取。

默认仍使用 Qt 默认皮肤。

## 核心指导代码

运动预览不应直接依赖某张图片路径：

```cpp
struct PreviewVisualAssets {
    QPixmap movingObject;
    QIcon playIcon;
    QIcon pauseIcon;
    QIcon resetIcon;
};
```

资源加载方向：

```cpp
// PreviewPanel 从 SkinProvider 获取 PreviewVisualAssets。
// 如果 SkinProvider 没有资源，则使用 Qt 默认按钮文本和简单几何图形占位。
```

## 注意

- 网格、坐标轴、曲线、采样点、控制点属于绘图功能，不使用图片换皮。
- 运动预览中的“物体外观”可以换皮。
- 业务数据和皮肤资源必须分离。
