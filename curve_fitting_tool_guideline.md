# 曲线绘制与拟合工具指导大纲

## 1. 项目定位

本工具用于生成和编辑物体随时间变化的运动曲线。用户可以通过两种方式创建曲线：

- 在曲线拟合区域直接绘制曲线或放置控制点，生成拟合公式和物体运动效果。
- 在滑块区域拖动滑块，录制指定时间内的运动过程，结束后自动生成拟合曲线。

工具输出的核心结果包括：

- 曲线图像
- 曲线公式
- 物体运动预览
- 总路程
- 当前速度
- 拟合误差或可信度提示

曲线方程不再限定为多项式，而是支持 SmoothStep、Logistic、指数趋近、低阶多项式，以及通过 PySR 进行自动符号回归。

## 2. 操作布局方案

界面采用三块主区域：

- 左上：曲线拟合区域
- 左下：滑块区域
- 右侧：结果展示区域

整体布局应接近用户提供的草图：

```text
+-------------------------------+---------------+
|                               |               |
|                               |               |
|        曲线拟合区域            |  结果展示区域  |
|                               |               |
|                               |               |
+-------------------------------+               |
|                               |               |
|        滑块区域                |               |
|                               |               |
+-------------------------------+---------------+
```

### 2.1 可拖拽分区要求

每一个模块都应支持拖拽调整尺寸。

必须支持：

- 曲线拟合区域和滑块区域之间有横向分隔条，可上下拖动。
- 左侧操作区和右侧结果展示区域之间有纵向分隔条，可左右拖动。
- 拖动分隔条时，区域内容实时重新布局。
- 每个区域都有最小宽度和最小高度，避免被拖到不可用。
- 拖动后的布局比例应保存，下次打开软件时恢复。
- 双击分隔条可恢复默认布局。

建议默认比例：

- 左侧操作区占总宽度约 70%。
- 右侧结果展示区域占总宽度约 30%。
- 曲线拟合区域占左侧高度约 65%。
- 滑块区域占左侧高度约 35%。

### 2.2 布局交互原则

拖拽布局时应保证：

- 曲线拟合区域中的网格、曲线、控制点自动适配新尺寸。
- 滑块区域中的滑轨和时间显示自动适配新尺寸。
- 结果展示区域中的公式、参数、误差、运动预览自动换行或滚动。
- 缩小窗口时优先保留曲线拟合区域的可用空间。
- 结果展示区域过窄时，复杂公式应折行或进入横向滚动。

## 3. 技术方向

### 3.1 主程序方向

主程序建议使用 C++ 制作，前端界面建议采用 Qt。

Qt 适合承载以下能力：

- 自定义曲线绘制区域
- 鼠标交互
- 网格显示与缩放
- 可拖拽分区布局
- 滑块录制
- 计时器
- 曲线实时刷新
- 公式展示
- 运动预览
- 参数面板

### 3.2 拟合方向

拟合系统建议采用混合方案：

- 低阶多项式：使用 C++ 本地拟合能力完成。
- SmoothStep：使用固定函数族参数化拟合。
- Logistic：使用固定函数族参数化拟合。
- 指数趋近：使用固定函数族参数化拟合。
- 自动拟合：使用 PySR 进行符号回归。

PySR 不建议直接嵌入 C++ 主程序，而是作为独立 Python 拟合服务或后台工作进程使用。

## 4. 支持的曲线类型

### 4.1 低阶多项式

适合表达普通平滑运动、简单加减速和局部趋势。

建议支持：

- 一次多项式
- 二次多项式
- 三次多项式
- 四次多项式
- 五次多项式

使用场景：

- 用户希望得到简单、可控、稳定的公式。
- 曲线变化不复杂。
- 需要快速实时拟合。

注意事项：

- 阶数过高容易产生震荡。
- 显示公式时小数最多保留 2 位。
- 内部计算应保留更高精度。

### 4.2 SmoothStep

适合表达平滑开始、平滑结束的运动。

使用场景：

- UI 动画
- 物体平滑插值
- 无突兀起停的运动

特点：

- 起点和终点速度自然趋近于 0。
- 公式简洁。
- 非常适合从起点移动到终点的固定时长动画。

建议提供：

- SmoothStep
- SmootherStep

### 4.3 Logistic

适合表达 S 形变化。

使用场景：

- 先慢后快再慢
- 带有饱和值的增长或移动
- 模拟缓慢启动和逐渐稳定的过程

特点：

- 曲线中段变化明显。
- 两端趋于平缓。
- 适合表现自然过渡。

注意事项：

- 需要约束参数范围，避免拟合出过陡或过平的曲线。
- 需要处理位置信息允许负值的情况。

### 4.4 指数趋近

适合表达快速接近目标值，然后逐渐稳定的运动。

使用场景：

- 阻尼式接近
- 缓动追踪
- 目标趋近
- 速度逐渐衰减

特点：

- 起始变化快，后期变化慢。
- 非常适合模拟惯性或追踪类效果。

注意事项：

- 需要明确目标值。
- 需要避免无限接近导致终点难以准确落位。

### 4.5 PySR 自动符号拟合

PySR 用于在用户没有指定曲线类型，或希望自动寻找简洁公式时使用。

使用场景：

- 用户绘制了复杂曲线。
- 用户录制了非标准运动。
- 固定函数族无法很好表达曲线。
- 希望自动发现较简洁的数学表达式。

建议限制 PySR 的搜索范围：

- 只允许常用运算。
- 限制公式复杂度。
- 限制表达式深度。
- 优先选择可读性较高的公式。

不建议：

- 每帧实时运行 PySR。
- 对所有拖动过程持续调用 PySR。
- 允许 PySR 生成过度复杂、难以解释的表达式。

## 5. 界面模块规划

### 5.1 曲线拟合区域

曲线拟合区域位于左上，是主要工作区。

必须支持：

- 绘制区域缩放
- 绘制区域平移
- 网格常显示
- 网格随缩放调整密度
- 坐标轴显示
- 曲线绘制
- 控制点放置
- 极值点放置
- 起始时间从 0 秒开始
- 末端点以最后一个放置点为准
- 速度允许为负数
- 曲线与横坐标形成的封闭区域高亮
- 总路程显示
- 当前时间显示
- 当前速度显示
- 有效范围显示
- 上述数据信息可折叠显示
- 拟合结果叠加显示
- 随模块尺寸变化自动重排绘图内容

交互建议：

- 鼠标左键放置或拖动点。
- 鼠标滚轮缩放绘制区域。
- 鼠标中键或空格拖动画布。
- 右键打开点操作菜单。
- 双击控制点可编辑数值。
- 拖动模块边界时，当前视图比例和中心点尽量保持不变。

视觉建议：

- 原始绘制轨迹和拟合曲线使用不同颜色。
- 曲线与横轴封闭区域使用明显半透明颜色。
- 极值点使用特殊标记。
- 起点和终点使用固定标识。
- 网格在任何缩放等级下保持可读。
- 当前时间、当前速度、总路程、有效范围建议做成类似 Blender 视图信息层的轻量浮层。
- 数据信息浮层应支持折叠、展开和固定显示，避免遮挡曲线操作。

### 5.2 滑块区域

滑块区域位于左下，用于通过拖动滑块生成一维运动数据。

滑块只表示一维位置变化，录制结果映射到曲线拟合区域中的纵轴位置信息。

必须支持：

- 鼠标按下滑块开始录制
- 鼠标松开滑块结束录制
- 拖动过程中持续采样
- 显示录制计时器
- 显示当前录制时间
- 显示滑块当前位置
- 设置直线区域总长度
- 录制结束后自动拟合
- 录制曲线同步显示到曲线拟合区域
- 区域高度变化后滑轨和状态信息自动适配
- 时间显示可折叠，采用类似 Blender 数据信息显示的轻量样式

录制规则：

- 一次完整鼠标拖动记录一段整体运动。
- 时间从按下滑块开始计算。
- 松开滑块后本次录制结束。
- 若用户重新录制，应询问是否替换当前曲线或追加为新曲线。

速度计算建议：

- 使用相邻采样点估算速度。
- 对速度结果进行平滑处理。
- 避免鼠标抖动导致速度尖峰。
- 速度结果不在滑块区域重点展示，而是在曲线拟合区域的信息浮层中显示。

### 5.3 结果展示区域

结果展示区域位于右侧，用于集中展示公式、拟合质量、运动预览和参数结果。

必须支持：

- 显示当前拟合类型
- 显示公式
- 小数最多保留 2 位
- 显示拟合误差
- 显示公式复杂度
- 显示运动预览
- 显示拟合状态
- 区域宽度变化后自动换行、折叠或滚动

建议分为以下子区：

- 公式区
- 参数区
- 拟合质量区
- 运动预览区
- 操作按钮区

建议支持：

- 复制公式
- 切换公式显示精度
- 显示导数公式
- 显示参数列表
- 播放、暂停、重置运动预览
- 拖动时间轴预览
- 多候选公式对比

公式显示规则：

- 面向用户时保持简洁。
- 内部计算不应受显示精度影响。
- 对 PySR 生成的复杂公式，应提供简化显示。
- 结果区过窄时，公式优先折行显示。

### 5.4 参数设置入口

参数设置可以放在结果展示区域内，也可以作为结果展示区域顶部的折叠面板。

建议包含：

- 曲线类型选择
- 多项式阶数
- 自动拟合复杂度
- 录制总时长限制
- 直线区域总长度
- 网格显示单位
- 曲线显示比例
- 是否显示速度曲线
- 是否显示原始采样点
- 是否显示拟合误差
- 是否显示封闭区域

曲线类型建议提供：

- 自动 PySR
- SmoothStep
- Logistic
- 指数趋近
- 低阶多项式

## 6. 拟合系统设计

### 6.1 拟合模式

拟合系统分为两类：

- 指定函数族拟合
- 自动符号回归拟合

指定函数族拟合适合用户已经知道目标曲线类型的情况。

自动符号回归拟合适合用户只提供数据，希望工具自动寻找表达式的情况。

### 6.2 拟合结果内容

每次拟合应生成一份完整结果：

- 曲线类型
- 公式文本
- 参数列表
- 有效时间范围
- 拟合误差
- 采样点数量
- 总路程
- 是否满足起点和终点
- 是否存在异常震荡

### 6.3 PySR 使用边界

PySR 只在以下情况调用：

- 用户选择自动拟合。
- 用户点击重新拟合。
- 绘制或录制完成后进行最终拟合。

PySR 不用于：

- 鼠标拖动中的实时预览。
- 每一帧刷新。
- 高频连续拟合。

### 6.4 拟合质量判断

拟合完成后应判断：

- 误差是否过大
- 公式是否过于复杂
- 曲线是否明显偏离采样点
- 起点和终点是否符合预期
- 是否出现不合理震荡
- 速度是否出现异常尖峰

当拟合质量较差时，工具应提示用户：

- 降低多项式阶数
- 改用 SmoothStep 或 Logistic
- 增加或调整控制点
- 重新录制
- 放宽或收紧自动拟合复杂度

## 7. 数据流规划

### 7.1 手绘曲线数据流

用户在曲线拟合区域操作后，数据流如下：

1. 用户绘制曲线或放置控制点。
2. 系统整理控制点。
3. 系统判断时间顺序和有效范围。
4. 根据当前曲线类型进行拟合。
5. 生成公式。
6. 绘制拟合曲线。
7. 计算速度和总路程。
8. 更新结果展示区域。
9. 更新运动预览。

### 7.2 录制曲线数据流

用户在滑块区域拖动滑块后，数据流如下：

1. 鼠标按下滑块。
2. 计时器启动。
3. 系统持续记录时间、位置和速度。
4. 鼠标松开滑块。
5. 计时器停止。
6. 系统清洗采样数据。
7. 系统平滑速度。
8. 根据当前拟合模式生成公式。
9. 在曲线拟合区域显示录制轨迹和拟合曲线。
10. 更新公式、总路程和运动预览。

### 7.3 PySR 数据流

PySR 拟合建议采用独立进程方式：

1. C++ 主程序整理采样点。
2. 主程序将拟合请求发送给 Python 工作进程。
3. Python 工作进程调用 PySR。
4. PySR 返回候选公式。
5. Python 工作进程筛选最佳结果。
6. Python 输出结构化拟合结果。
7. C++ 主程序读取结果。
8. 主程序显示公式并驱动运动预览。

## 8. 坐标与物理含义

必须提前明确纵轴含义。

建议优先采用：

- 横轴：时间，单位为秒。
- 纵轴：位置。

在这种定义下：

- 曲线值表示物体位置。
- 滑块录制的是一维位置随时间的变化。
- 速度是曲线对时间的导数。
- 总路程是速度绝对值随时间的积分。

第一版固定使用“纵轴为位置”的方案，不切换为速度轴。这样更直观，也更适合滑块录制。

## 9. 总路程与封闭区域

### 9.1 总路程

当曲线表示位置时：

- 总路程应根据速度绝对值计算。
- 不能简单使用终点位置减起点位置。
- 如果曲线来回运动，总路程会大于净位移。

### 9.2 封闭区域

曲线与横坐标形成的封闭区域需要明显着色。

注意：

- 曲线在横轴上方和下方都应正确填充。
- 曲线穿越横轴时应分段处理。
- 填充颜色应明显，但不能遮挡网格和曲线。

封闭区域可用于辅助理解曲线趋势，但它不一定等于总路程。

## 10. 关键风险

### 10.1 布局风险

- 用户可能把某个区域拖得过小，导致内容不可用。
- 结果区公式过长时可能挤压其他信息。
- 曲线区尺寸变化后坐标比例可能混乱。

应对方式：

- 设置每个区域的最小尺寸。
- 结果展示区支持滚动和折叠。
- 曲线区重绘时保持坐标中心和缩放比例。
- 提供一键恢复默认布局。

### 10.2 手绘曲线风险

- 用户可能绘制出一个时间对应多个位置的曲线。
- 控制点顺序可能混乱。
- 极值点可能与拟合结果不一致。

应对方式：

- 限制曲线按时间方向生成。
- 自动排序控制点。
- 对非法点进行提示。
- 将极值点作为约束或参考点，并在界面中明确区分。

### 10.3 拟合风险

- 多项式阶数过高会震荡。
- PySR 可能生成过复杂公式。
- Logistic 或指数趋近可能无法拟合某些曲线。
- 采样噪声会影响速度计算。

应对方式：

- 限制阶数。
- 限制 PySR 复杂度。
- 显示拟合误差。
- 对采样数据进行平滑。
- 提供手动切换函数类型能力。

### 10.4 性能风险

- PySR 拟合可能耗时较长。
- 大量采样点会影响绘制性能。
- 实时拟合可能造成界面卡顿。

应对方式：

- PySR 放在后台进程。
- 实时阶段只做轻量预览。
- 拟合结束后再更新最终公式。
- 对采样点进行降采样。

## 11. 建议实施阶段

### 第一阶段：可拖拽布局与基础绘图

目标：

- 完成三块主区域布局。
- 完成横向和纵向分隔条拖拽。
- 完成布局比例保存与恢复。
- 完成曲线拟合区域的网格、缩放、坐标转换。

验收标准：

- 用户可以拖动每个模块的边界调整尺寸。
- 重启软件后能恢复上次布局。
- 曲线区缩放后曲线和网格保持正确。

### 第二阶段：基础绘制与本地拟合

目标：

- 支持控制点放置。
- 支持曲线绘制。
- 支持低阶多项式拟合。
- 显示公式。
- 在结果展示区域显示拟合结果。

验收标准：

- 用户可以放置点并看到拟合曲线。
- 公式可以正确显示。
- 结果展示区域能显示公式、误差和基本参数。

### 第三阶段：录制滑块

目标：

- 完成滑块区域。
- 支持按下开始、松开结束。
- 记录时间、位置和速度。
- 录制结束后生成曲线。

验收标准：

- 拖动滑块可以生成采样轨迹。
- 计时器和速度显示正常。
- 录制曲线能同步显示到曲线拟合区域。

### 第四阶段：多函数族拟合

目标：

- 增加 SmoothStep。
- 增加 Logistic。
- 增加指数趋近。
- 增加曲线类型选择。

验收标准：

- 用户可以选择不同函数类型。
- 不同函数类型能生成对应公式。
- 运动预览与公式一致。

### 第五阶段：PySR 自动拟合

目标：

- 接入 Python 工作进程。
- 接入 PySR。
- 支持自动符号回归。

## 12. 推荐优先级

高优先级：

- 三块主区域布局
- 每个模块可拖拽调整尺寸
- 布局比例保存与恢复
- 曲线区缩放和网格
- 控制点绘制
- 滑块录制
- 低阶多项式拟合
- 公式显示

中优先级：

- SmoothStep
- Logistic
- 指数趋近
- 总路程
- 速度曲线
- 封闭区域填充
- 运动预览

低优先级：

- PySR 自动拟合
- 复杂度调节
- 多候选公式对比
- 导出功能
- 高级约束拟合

## 13. 最终建议

第一版应先完成用户草图中的三块区域，并保证每个模块都能拖拽调整尺寸。布局稳定之后，再实现曲线绘制、滑块录制和本地低阶拟合。

PySR 建议作为后期自动拟合能力接入，不要放在第一阶段。这样可以先保证工具的基本操作体验稳定，再逐步增加复杂拟合能力。
